CSS por sus siglas en inglés significa: Cascading Style Sheets u Hojas de Estilo en Cascada. Ni HTML ni CSS son lenguajes de programación, HTML es un lenguaje de marcado que define contenido y CSS es un lenguaje de definición de estilos que le van a dar presentación a nuestros Documentos HTML. CSS aunque como tal no es lenguaje de programación, pero agrega caracteristicas que son propias de los lenguajes de programación, por ejemplo en este Lenguaje tenemos las Coustom Properties o variables CSS que permiten almacenar un valor, también ciertas funciones que nos permiten aplicar ciertos comportamientos y ciertos atributos que permiten programar algunas cosas (ej animación).

Sintáxis CSS

Una regla CSS consta de dos partes:
  1. Selector: Que puede ser cualquier etigueta HTML.
  2. Bloque de declaraciones.
El Selector más básico son las etiquetas que, cuando aplicamos estilos a un selector, los estilos aplicaran en todas esas etiquetas del Documento.
Los bloques de declaraciones tiene la siguiente sintáxis; "Atributo: Valor;". Poner dos puntos y punto y coma es imprecindible, de otra manera nos dará un error de sintáxis. Si el nombre del atributo tiene más de dos palabras, se escribiran seguidas de un guión (una-palabra-despues-de-otra). Los bloques de declaración van entre llaves.

Comentarios en CSS

Para comentar en CSS se usa al igual que en Javascript, diagonal asterisco para abrir y asterisco diagonal para cerrar (/* comentario */). Con los atajos del teclado podemos seleccionar un bloque de líneas y comentarlas en un solo paso. (shift + alt + a).

Formas para aplicar Estilos

Hay tres formas:

  • Estilos en linea
  • Estilos internos
  • Definición de Hoja Externa
La forma de definir estilos en línea es a través del atributo "style" como atributo de HTML.
Definir estilos internamente es a través de abrir etiquetas "style" (que se hace en la cabecera del Documento).
La tercer forma es la ideal, se define una hoja externa, que contiene su propio archivo CSS y se vincula al Documento HTML a través de la etiqueta "link", con los atributos de rel y valor "stylesheet" y el atributo "href" al cual indicamos la ruta hacia la hoja externa.
La buena práctica dicta que es mejor siempre usar una hoja de estilos externa, pero hay veces que por practicidad o alguna necesidad puntual, se pueden usar las otras formas para aplicar los estilos.

Hay otras formas de llamar una hoja de estilo, y además, desde una hoja podemos vincular otra hoja con arroba import e invocar una función CSS lamada url "@import url()", dentro de la función indicamos la ruta de la hoja que estamos invocando. El problema con este método es el siguiente: cuando el navegador va descargando un Documento y lee una etiqueta link, comienza a descargar los estilos CSS, pero continua leyendo y descargando el Documento HTML. Si el navegador lee un @import, se detiene ahí hasta completar la lectura y descarga de este contenido, lo que puede perjudicar en el rendimiento general (comportamiento bloqueante).

Selectores Básicos

Un selector es cualquier etiqueta HTML a la que le vamos a aplicar estilos.
Existen tres tipos de selectores básicos: Etiquetas, Identificadores y Clases.

  • Etiquetas - Son elementos HTML
  • Identificadores - id - #
  • Clases - atributo class - .class

Dar estilos con id, se considera una mala práctica, pues solo podemos tener un identificador (es como una clave, el dni de algo en particular). Una de las caracteristicas de CSS es la re utilización de los estilos, y con los id no lo podemos hacer. los id son más utiles a la hora de interactuar con JavasCript.
Las clases son la manera ideal de maquetar y dar estilos, pues están entre medio. Las etiquetas HTML son muy amplias y le dan estilo a todo el Documento, los id son identificadores únicos, entonces las clases nos dan un equilibrio deseado, permitiendo reutilizar código. Es muy ùtil dar nombres alusivos a los estilos que aplicamos, porque eso facilita la reutilizaciòn y comprension de los mismos.

Selectores de Hijos

Selector de Hijos Directos

Un hijo directo es el elemento que está en el primer nivel. En una ul (lista desordenada), los hijos directos son cada li (list item). Para aplicar estilos a los hijos directos se usa el símbolo mayor que (>) en el nombre de la clase, luego se escribe la etiqueta a la cual se van a aplicar los estilos.

  • Primavera
  • Verano
  • Otoño
  • Invierno

Selector de Hijos Descendientes

Los estilos para los hijos descendientes, es decir: de un segundo nivel, se aplican decando un espacio en blanco, y escribiendo el nombre de la etiqueta. En el primer caso se había aplicado el color de fondo en las etiquetas li, por eso se coloreo todo, pues son etiquetas de bloque, en el segundo caso en cambio solo se colorean las palabras, pues los estilos se aplicaron a las etiquetas b, que son de línea.

Selectores de hermanos

Selectores de Hermanos en General

Este tipo de selector se aplica directamente a todos los hermanos, se escribe la clase a la cual se quiere aplicar los estilos y después de un espacio se escribe una tilde (~) seguido del selector al cual queremos aplicar. Los estilos se aplicarán a partir del elemento que hayamos seleccionado como referencia.

  • Verano
  • Otoño
  • Invierno

Selectores de Hermanos Adyacentes

Para Hermanos adyacentes hacemos el mismo procedimiento, pero el símbolo que distingue a estos es la suma (+), y solo se aplicará al elemento que le sigue al que tomamos como referencia.

  • Primavera
  • Verano
  • Otoño
  • Invierno

Este tipo de slector aplica para todas las etiquetas y elementos, no tan solo a las clases.

Selectores de Atributos

Los selectores de atributos nos van a permitir aplicar estilos tomando como referencia cualquier atributo (por ejemplo un href). Tienen una gran potencia con diferentes variantes. Entre corchetes ponemos el atributo al cual queremos poner como referencia.
Usamos el asterisco (*) que funciona como un comodín que aplica el estilo si contiene la palabra en cuestion.
Cuando usamos el símbolo sircunflejo (^) el estilo se va a aplicar siempre y cuando empiecen con la palabra o termino que querramos aplicar.
El símbolo de dolar ($) aplica el estilo que contiene el texto requerido al final.
El símbolo de la tilde (~) además de servir para los hermanos, si está dentro del corchete servirá también para buscar si existe un término dentro de una lista de clases separada por espacios.
El símbolo pipe o pleca (|) es similar al anterior (~), funciona de la misma forma, solo que al final el término o texto en cuestión debe terminar con un guión medio(-).

Hay pseudoclases para elementos de tipo formulario, para enlaces. Hay una jerarquía, por lo cual importa el orden en que se definien, siendo primero :link, :active, :visited y :hover.

Algoriitmo en CSS

El algoritmo en CSS es la manera en que el navegador va aplicando los estilos al documento HTML. Es vital este concepto para entender como se aplican y en algunas ocasiones, se sobreescriben las reglas de CSS. Las reglas básicas del algoritmo son tres y son las siguientes:

  1. La Cascada
  2. La Espeficidad
  3. La Herencia

La Cascada

La cascada es el mecanismo que tiene el navegador Web para ir aplicando los estilos. En base a esto toma tres aspectos en particular, que va aplicando en un orden de jerarquía;
1- Origen del código; Primero el User Agent que son los estilos que trae por defecto el navegador. Segundo las personilizaciones que el usuario le hace a su navegador; y Tercero viene los estilos CSS que el autor define en su Documento HTML o "Estilos del Autor". Después el navegador va leyendo línea por línea, emepezando a plicar los estilos y cambiendolos conforme va leyendo una nueva instrucción.
2- Especifidad; Esta viene muy asociada con la Cascada, es le peso que tiene cada selector (como un valor o puntaje). El estilo que queda es el que mayor peso tiene y sobreescribe por tanto a los estilos que se hayan definido con menos valor. Si dos selectores tienen un mismo peso, el estilo que se aplica es el último (por cascada). la especifidad se mide en cuatro digitos separados por comas y de derecha a izquierda y con valores que van desde cero, aumentando su peso, conforme el número de la izquierda es mayor.

Practicando cascada y especifidad

Practicando en la hoja de estilos lo aprendido sobre cascada y especifidad en la siguiente cita:

Se tolerante con los demás y estricto contigo mismo. Marco Aurelio

En primer paso se aplica un estilo a la etiqueta blockuote, en segundo a la misma etiqueta, pero asociada a la clase ".cita-marco"; en trecer solo a la clase (queda como estilo el anterior porque tiene mayor especifidad). En cuarto lugar se aplica estilo al identificador, pero a propósito se lo mezcla arriba en la cascada, y quedan los estilos de este porque es el que tiene más especifidad. por último se aplica un estilo a la etiqueta asociada a un identificador, quedando tanto por especifidad como por cascada.

Para demostrar que la especifidad rompe la cascada, al principio de las definiciones de estilos, se pone una etiqueta asociada al identificador y a la clase, y quedan estos estilos porque por más que esté al principio de la cascada, su especifidad es mayor a todos. pero como se terminó agregando una etiqueta style al mismo documento HTML, terminó aplicando los estilos pues tiene la mayor especifidad. Important rompe con la especifidad y con la cascada. En caso de dos !important, quedan los estilos del último por la cascada.

La Herencia

La Herencia es la capacidad de un selector de obtener (heredar) los valores de sus ancestros más cercanos, para aplicarla se usa el valor "inherit" (heredar), si queremos evitarla, podemos asignar otro valor o inicializarl la propiedad en cuestion con el valor "initial". Hay propiedades que no se heredan, por ejemplo bordes, fondos. La tipografía si se hereda

Se tolerante con los demás y estricto contigo mismo. Marco Aurelio

Cuando usamos la misma cita de Marco Aurelio, primeros definimos una color de fondo y un tamaño distinto, y después con el valor inherit, volvemos a los valores heredaros por su etiqueta padre "blockquote". si uno varía por ejemplo el tamaño de letra de la etiqueta padre, el hijo "heredará" por defecto dicha propiedad.

Prefijos de los Navegadores

HTML, CSS y JavaScripts son standars de los navegadores, por lo tanto no los debemos descargar en software para que funciones, son standar libres que viven en ellos. Los prefijos de los navegadores son usados para tecnologías experimentales y se identifican por alguna sigla o abreviatura en particular.

  • Navegadores microsoft -ms-
  • Firefox -moz-
  • Todos los demás Navegadores -webkit-

De esta forma quedarían en la hoja de Estilos:
-webkit-user-select: none; para el comun de los Navegadores.
-moz-user-select: none; para firefox
-ms-user-select: none; para navegadores microsoft
user-select: none; cuando no necesitas poner prefijos.
